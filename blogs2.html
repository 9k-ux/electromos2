Introduction

  • Arduino is an open-source electronics platform based on easy-to-use hardware and software.

    • The software provided by Arduino gives a level of of abstraction (via functions and libraries) which helps us avoid directly interacting with the underlying hardware.

  • Arduino boards can do a lot of stuff including :

    • To read inputs - Light on a sensor, a finger on a button, or a Twitter message.

    • Turn it into outputs - Activating a motor, turning on an LED, publishing something online.

  • You can tell your board what to do by sending a set of instructions to the micro-controller on the board.


Components Overview

In this part we will be describing an Arduino Uno Board


GPIO Pins

  • GPIO pins, also called as General Purpose Input /Output Pins, are a combination of Digital Pins and Analog Pins, Serial communication Pins Power and USB.

    • 14 Digital Pins (6 PWM).

      • 6 Analog Pins (These are basically digital pins with ADC Support).

  • These pins are used to give Input to the board like switch state or sensor data.

  • Also used to show outputs like controlling LEDs, showing text etc.


Crystal Oscillator

  • This Component helps the board in time calculation. Playing a major role in sending and receiving data from other devices.

  • The frequency at which it works indicates the speed of the board in making decisions.

  • The frequency is generally in the range of about 16MHz.


Voltage Regulator

  • This plays a major role in safety of the board by regulating the amount of voltage to prevent overheating the board.

  • It delivers only required amount of voltage to the input of the Arduino board and controls the DC power supply utilized by the microcontroller and other peripheral devices.


Microcontrollar

  • This is the brain of the board. It uses AtMega328p, an 8-bit microcontroller.

  • These are pre-programmed containing a bootloader which takes up 0.5 KB of the Flash Memory (32 KB) that simplifies uploading program.

Aurdino IDE

  • An Integrated Development Environment (IDE).


  • Arduino IDE is a spin-off of Processing IDE (an open-source IDE with a Java-based programming language) which is used to write functions in C/C++ and upload it onto the Arduino compatible boards to execute.

  • It has Wiring Framework Library (a C/C++ library for micro-controller programming) using which one can write common input/output operations over a micro-controller easily.

  • Download and install from the link → Arduino IDE .

    • Note: After installing the IDE be sure to choose the right board in 'Tools→Board'. Connect the board to the PC using a USB Cable.

  • The routine to implement any project is to first write code on the IDE, then setup hardware connections, finally upload your code to the hardware.


Sketch

  • A sketch is the name that Arduino uses for a program. It's the unit of code that is uploaded to and run on an Arduino board.

  • Arduino programming language can be divided in three main parts.
    • Values - Variables and Constants.

    • Structure - setup() and loop()

    • Functions - Library Functions, Custom Functions.


  • The sketch is stored using .ino extension (Referring to Arduino).

  • Let's cover some basic syntax, semantics of an Arduino Sketch in the below section.

Comments

  • Comments are used for documenting the code - to explain what the program does, how it works, or why it's written the way it is.

  • The documentation can be useful for both a person reading the code or the person who wrote the code to recollect what the code does later.

Multi-line Comment

  • Everything between the /* and */ is ignored by the Arduino when it runs the sketch.

/* This is a multiline comment.
  * Line 1.
  * Line 2.
  * Line 3.
  */

The * at the start of each line is only there to make the comment look pretty, and isn't required.

Single Line Comment

  • There's another style for short, single-line comments. These start with // and continue to the end of the line.

// This is a single line comment.

  • A variable is a place for storing a piece of data. It has the following components :

    • A name - A placeholder for the variable.

    • A type - int, float, char, bool, byte etc.

    • A value - Value to be stored inside the variable in accordance with the type.

  • Ex - Declare and define a variable to indicate which Arduino Pin Number we want an LED to be connected to.
    • Declare a variable with the name ledPin, the type int, and an say initial value of 13.

      • We generally use Digital Pin 13 as an LED Pin Output because it already has an LED and resistor attached to it that's soldered to the board on most Arduino Boards.

      int ledPin = 13;


    • Every time the name ledPin appears in the code, its value 13 will be retrieved.

    • In this case, the person writing the program could have chosen not to bother creating the ledPin variable and instead have simply use 13 everywhere they needed to specify a pin number.

    • The advantage of using a variable is that it's easier to move the LED to a different pin - you only need to edit the one line that assigns the initial value to the variable.

Structure

  • Arduino Programming Language is based on Wiring Library Framework - a C/C++ library which makes common input/output operations over a micro-controller much easier.

  • Hence as per the Wiring Framework, a basic Arduino Sketch requires a minimum structure consisting only two functions:

    • setup()
    • loop()
  • You need to include both functions in your sketch, even if you don't need them for anything.

setup()

  • It is a function that is run once at the start of a program which can be used to define initial environment settings.

  • It's a good place to do setup tasks like setting pin modes or initializing libraries.

loop()

  • It is a function that is called repeatedly until the board is powered off or reset.

  • This acts like the heart of the most sketches where we write most of the semantics of the program in.

        • void setup(){

        }

        void loop(){
          //Main code goes here
        }

  • You can call a function that's already been defined (either in your sketch or as part of the Arduino language Library) using the function's function_name(parameters).

Library Functions

  • Arduino Language Provides some in-built functions using the Wiring Framework Library.

  • Some of the most commonly used functions like pinMode() digitalWrite() delay() etc., will be explored in the further sections of this blog.

Custom Functions

  • Arduino Programming also allows defining custom functions similarly to C/C++.

  • We just need to use the above syntax for defining the function and write what the function does in the body of the function.

  • Ex : A function multiply(x,y) to multiply two numbers - x and y.

void setup(){
  Serial.begin(9600);
  int i = 2;
  int j = 3;
  int k;
}

void loop() {
  k = multiply(i, j); // k now contains 6
  Serial.println(k);
  delay(500);
}

//Returns the Product of the given parameters - i and j
int multiply(int x, int y){
  int result;
  result = x * y;
  return result;
}

  • Follow the below circuit schematics and provide power.

    • Arduino Board must be connected to the Power Source (In our case PC) throughout run time.

Sketch

/*
  Blink
  
  Turns an LED on for one second, then off for one second, repeatedly.
  
  Most Arduinos have an on-board LED you can control. On the UNO, MEGA and ZERO
  it is attached to digital pin 13, on MKR1000 on pin 6. LED_BUILTIN is set to
  the correct LED pin independent of which board is used.
  If you want to know what pin the on-board LED is connected to on your Arduino
  model, check the Technical Specs of your board at:
  https:// www.arduino.cc/ en/Main/ Products
  
  modified 8 May 2014
  by Scott Fitzgerald
  modified 2 Sep 2016
  by Arturo Guadalupi
  modified 8 Sep 2016
  by Colby Newman
  
  This example code is in the public domain.
  
  https:// www.arduino.cc /en/ Tutorial/ BuiltInExamples/ Blink
*/

// the setup function runs once when you press reset or power the board
void setup() {
  // initialize digital pin LED_BUILTIN as an output.
  pinMode(LED_BUILTIN, OUTPUT);
}

// the loop function runs over and over again forever
void loop() {
  digitalWrite(LED_BUILTIN, HIGH); // turn the LED on (HIGH is the voltage level)
  delay(1000); // wait for a second
  digitalWrite(LED_BUILTIN, LOW); // turn the LED off by making the voltage LOW
  delay(1000); // wait for a second
}

  • pinmode() - Tells the Board to set the given Pin to either INPUT or OUTPUT.

  • digitalWrite() - Tell the Board to set the given Digital Pin Voltage Level to HIGH or LOW.

  • delay() - Tells the board to do nothing for given amount of milliseconds.
